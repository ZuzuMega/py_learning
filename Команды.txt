# Команды cmd

	dir  - показать файлы в папке 
	cls - очистить консль 
	cd.. - вернуться в предыдущую папку
	mkdir $folder_name - создать новую папку


# Команды git
	
	git init - инициализация git в папке проекта
	git status - статус проекта	

	git add $file_name - гит начнет отслеживать изменения в файле
	git add . - отслеживать все файлы в папке
	git rm --cached $file_name - гит перестанет отслеживать файл

	git commit -m "$commit_name" - делаем коммит - фиксируем текущую версию, чтобы ее запушить или восстановить

	git branch - показать ветки проекта
	git branch $branch_name - создать новую ветку с названием $branch_name
	git branch -D $branch_name - удалить ветку 
	git checkout $branch_name - переключиться на ветку
	git checkout -b $branch_name - создать и сразу переключиться на новую ветку
	git merge $branch_name - объединить текущую ветку с веткой $branch_name

	git config --global user.name - имя, которое отобразится в git
	git config --global user.email - email, который отобразится в git
	git config --global -l - посмотреть текущий email и user

	git remote add origin $link_on_the_repos - размещение проекта в GitHub репозитории
	git push -u origin master - пуш проекта в репозиторий GitHub
	git push - пуш изменений в GitHub

	git clone $link_on_the_repos - копировать репозиторий себе на машину
	git pull - скачать актуальный проект с git на локальную машину 

	## Комментарии
	* Если в файле было изменение, то перед коммитом, с файлом нужно сделать команду git add ./$file_name
	* Для игнорирования файлов гитом (исключить файлы из отслеживания и коммита) нужно создать файл с названием ".gitignore" и в нем указать названия файлов, которые нужно игнорировать.
	* Для игнорирования папок в .gitignore можно указать /$folder_name
	* Для того, чтобы в ветку master добавить файлы из другой ветки, нужо перейти в master и выполнить команду merge. Файлы из другой ветки скопируются в master
	* Имя пользователя можно изменить. Для этого нужновы полнить команду git config --global user.name $user_name. Такой же командой можно изменить email
	* В GitHub требуется создать репозиторий, локально, через cmd выполняется привязка к аккаунту и выполняется пуш проекта в GitHub

	## Стандартная схема для нового проекта:
	1. git init - инициализировать git в проекте
	2. git add . - добавить файлы для отслеживания
	3. git commit -m "$commit_name" - создать коммит - зафиксировать версию проекта
	4. git push - отправить текущую версию в репозиторий
